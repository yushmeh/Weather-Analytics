## 🚀 Ключевые особенности

### 📡 Получение и хранение данных
- **Open-Meteo API** — бесплатный источник погодных данных, не требует API-ключа.
- Запрашивает температуру (макс/мин) за последние **7 дней** для Москвы.
- Сохраняет данные в локальную **SQLite** через `UPSERT` — повторный запуск обновляет записи без дублирования.

### 📊 Аналитика и визуализация
- **5 метрик** — средняя макс/мин температура, самый тёплый и холодный день, температурный размах.
- **Plotly-график** — двухпанельный интерактивный график сохраняется в `weather_chart.html`.
- **Полноценный дашборд** — `weather_dashboard.html` открывается без запуска Python и работает автономно.

### 🗂 Архитектура скрипта
Каждая функция отвечает ровно за один шаг пайплайна — от запроса к API до построения графика:

```
main.py
│
├── fetch_weather_data()       GET-запрос к Open-Meteo API
├── parse_weather_records()    Парсинг JSON → список кортежей
│
├── get_connection()           Подключение к SQLite
├── create_table()             CREATE TABLE IF NOT EXISTS weather_log
├── save_records()             INSERT OR REPLACE (UPSERT)
├── load_records()             SELECT всех записей
│
├── calculate_analytics()      Расчёт метрик из списка записей
├── print_analytics()          Вывод в консоль (таблица + метрики)
│
└── build_chart()              Plotly-график → weather_chart.html
```

## 🛠 Инструкция по запуску

Для работы необходим **Python 3.14+** и доступ в интернет.

```bash
# 1. Перейти в папку проекта
cd weather-analytics

# 2. Создать виртуальное окружение (рекомендуется)
python -m venv .venv

# Windows:
.venv\Scripts\activate
# macOS / Linux:
source .venv/bin/activate

# 3. Установить зависимости
pip install requests plotly
```

> `sqlite3` входит в стандартную библиотеку Python — устанавливать отдельно не нужно.

```bash
# 4. Запустить скрипт
python main.py
```

После запуска:
- аналитика выводится в консоль
- график сохраняется в `weather_chart.html`
- дашборд сохраняется в `weather_dashboard.html`

## 📐 Аналитические метрики

| Метрика | Описание |
| :--- | :--- |
| Средняя максимальная температура | Среднее `temp_max` за 7 дней |
| Средняя минимальная температура | Среднее `temp_min` за 7 дней |
| Самый тёплый день | Дата с наибольшим `temp_max` |
| Самый холодный день | Дата с наименьшим `temp_min` |
| Температурный размах | `max(temp_max) − min(temp_min)` за весь период |

Пример вывода в консоль:

```
─────────────────────────────────────────────
  📊  АНАЛИТИКА ПОГОДЫ ЗА 7 ДНЕЙ (Москва)
─────────────────────────────────────────────
  Средняя максимальная температура : +21.5 °C
  Средняя минимальная температура  : +13.3 °C
─────────────────────────────────────────────
  🌡️  Самый тёплый день  : 2026-05-21  (+31.2 °C)
  🥶  Самый холодный день: 2026-05-27  (+7.7 °C)
─────────────────────────────────────────────
  📐  Температурный размах за период: 23.5 °C
─────────────────────────────────────────────
```

## 🗄 Схема базы данных

База данных `weather.db` содержит одну таблицу:

```sql
CREATE TABLE IF NOT EXISTS weather_log (
    date     TEXT PRIMARY KEY,   -- формат YYYY-MM-DD
    temp_max REAL NOT NULL,      -- максимальная температура за день, °C
    temp_min REAL NOT NULL       -- минимальная температура за день, °C
);
```

Поле `date` — первичный ключ. Повторный запуск не создаёт дубликаты, а обновляет существующие записи через `INSERT OR REPLACE`.

## 📊 Дашборд

Файл `weather_dashboard.html` — автономный интерактивный дашборд, не требующий запуска Python:

- **Тёмная / светлая тема** — переключатель в шапке страницы
- **5 карточек метрик** — avg max/min, самый тёплый/холодный день, температурный размах
- **График Chart.js** — линии макс/мин с заливкой и столбцы диапазона на отдельной оси
- **Интерактивная таблица** — сортировка по любому столбцу, визуальные полоски диапазона, бейджи для крайних значений
- **Адаптивность** — корректно отображается на мобильных устройствах
- **Анимации** — плавное появление карточек и строк таблицы при загрузке

Чтобы подключить дашборд к реальным данным из `weather.db`, добавьте в `main.py` генерацию HTML с вставкой данных:

```python
import json

records_json = json.dumps([
    {"date": r[0], "temp_max": r[1], "temp_min": r[2]}
    for r in db_records
])
# Замените строку в weather_dashboard.html:
# const weatherData = <сюда вставить records_json>;
```

## ⚠️ Обработка ошибок

| Ситуация | Поведение |
| :--- | :--- |
| Нет интернета | Сообщение об ошибке + завершение |
| Таймаут API (> 10 с) | Сообщение об ошибке + завершение |
| Неверный HTTP-статус | Вывод кода ошибки + завершение |
| Неожиданная структура JSON | Указание отсутствующего ключа + завершение |
| Ошибка SQLite | Подробное сообщение + завершение |
| Пустой ответ API | Сообщение об ошибке + завершение |
